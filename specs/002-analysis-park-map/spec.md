# Feature Specification: Analysis Park Map Selector

**Feature Branch**: `002-analysis-park-map`  
**Created**: 2026-03-31  
**Status**: Draft  
**Input**: User description: "我要在分析页面添加地图功能。地图区域布局在顶部固定横栏下方，横向展示，适配网页宽度。功能：地图框右上角设置下拉选择框，内置预设市政公园选项列表（深圳湾公园、深圳东湖公园、笔架山公园、仙湖植物园）。选择公园后，地图自动定位到该地点并添加标记点。"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 查看分析页地图总览 (Priority: P1)

作为访客，我希望在分析页面顶部导航栏下方直接看到地图区域，并且地图横向铺满可用页面宽度，这样我进入页面后就能立即理解当前分析区域的位置语境。

**Why this priority**: 地图区域是本功能的入口能力；若位置和布局不正确，后续园区选择和定位体验都无法成立。

**Independent Test**: 打开分析页面并检查顶部固定横栏下方是否立即出现地图区域，且地图在目标移动宽度范围内无横向溢出。

**Acceptance Scenarios**:

1. **Given** 用户位于分析页面，**When** 页面完成加载，**Then** 地图区域显示在顶部固定横栏下方。
2. **Given** 用户在推荐移动视口宽度内访问页面，**When** 地图区域渲染完成，**Then** 地图横向适配页面宽度且无需水平滚动。

---

### User Story 2 - 通过公园列表切换定位 (Priority: P2)

作为访客，我希望通过地图右上角下拉框快速选择预设市政公园，并让地图自动定位并打点，这样我能一键切换到关注园区。

**Why this priority**: 公园选择与自动定位是该地图功能的核心交互价值。

**Independent Test**: 在分析页面依次选择“深圳湾公园、深圳东湖公园、笔架山公园、仙湖植物园”，确认每次选择后地图中心和标记点均切换到对应公园。

**Acceptance Scenarios**:

1. **Given** 地图区域已显示，**When** 用户打开下拉框，**Then** 列表包含深圳湾公园、深圳东湖公园、笔架山公园、仙湖植物园四个预设选项。
2. **Given** 用户选择任一公园，**When** 选择动作完成，**Then** 地图自动移动到该公园并显示对应标记点。
3. **Given** 用户从一个公园切换到另一个公园，**When** 新选择生效，**Then** 页面不发生整页刷新且可见状态与所选公园一致。

---

### User Story 3 - 异常与高频操作下保持一致性 (Priority: P3)

作为访客，我希望在网络波动或快速连续切换选择时，地图状态依旧明确、可恢复且最终结果正确，这样我不会因为短时异常失去操作信心。

**Why this priority**: 稳定的一致性反馈可降低误判和重复操作，提升地图交互可用性。

**Independent Test**: 连续快速切换多个公园并在弱网环境下观察结果，确认最终地图状态与最后一次选择一致，并有清晰失败提示。

**Acceptance Scenarios**:

1. **Given** 用户连续快速选择多个公园，**When** 地图更新完成，**Then** 最终显示必须匹配最后一次选择的公园。
2. **Given** 地图底图暂时不可用，**When** 用户进入分析页面，**Then** 用户仍能看到下拉选择控件和可理解的提示信息。

### Edge Cases

- 用户重复选择当前已选公园时，地图状态保持稳定且不产生重复标记。
- 用户在短时间内连续多次切换公园时，最终状态必须以最后一次选择为准。
- 地图数据加载失败或网络波动时，页面应提供清晰提示并允许用户继续选择园区。
- 在推荐宽度外的窄屏或宽屏条件下，地图区域仍保持可用且不引入横向滚动。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统 MUST 在分析页面顶部固定横栏下方展示地图区域。
- **FR-002**: 地图区域 MUST 以横向形式适配当前页面可用宽度，并保持可用可读。
- **FR-003**: 地图区域右上角 MUST 提供公园选择下拉控件。
- **FR-004**: 下拉控件 MUST 内置且仅内置以下公园选项：深圳湾公园、深圳东湖公园、笔架山公园、仙湖植物园。
- **FR-005**: 系统 MUST 在初始进入地图时为用户提供默认园区定位与标记点展示。
- **FR-006**: 用户选择任一公园后，系统 MUST 自动将地图定位到该公园并显示对应标记点。
- **FR-007**: 在任意时刻，地图中心位置、标记点与下拉框当前选择 MUST 保持一致。
- **FR-008**: 公园切换 MUST 在当前页面内完成，不得触发整页刷新。
- **FR-009**: 地图及其交互区域 MUST 保持纵向页面滚动体验，不得引入横向滚动依赖。
- **FR-010**: 当地图暂时不可用时，系统 MUST 显示可理解的提示信息并保持下拉选择功能可用。
- **FR-011**: 对于快速连续选择行为，系统 MUST 以最后一次用户选择作为最终展示结果。
- **FR-012**: 每次公园切换后，系统 MUST 在地图上维持单一有效标记点，避免重复标记造成歧义。

### API, UI, and Visualization Contract *(mandatory for web features)*

- **API-001**: 本特性不新增 Route Handler；园区清单与定位信息由前端内置预设数据提供。
- **UI-001**: 需要更新分析页面的地图展示区（位于顶部固定横栏下方）、地图右上角下拉选择区、以及随选择变化的定位标记显示区。
- **UI-002**: 需覆盖以下状态：默认园区展示状态、切换中状态、切换完成状态、地图不可用提示状态，并保证各状态在移动宽度下无横向溢出。
- **CHART-001**: `N/A`。本特性为地理地图定位与标记交互，不属于图表可视化需求。

### Key Entities *(include if feature involves data)*

- **ParkOption**: 预设市政公园选项，包含园区名称与用于定位的地理位置信息。
- **MapViewState**: 当前地图视图状态，包含当前选中公园、地图中心位置、当前标记点与可用性状态。
- **SelectionState**: 当前下拉选择状态，包含当前值、最后一次有效选择以及切换中的临时状态。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 在 375px 至 430px 推荐移动宽度中，100% 验证样本可在分析页顶部横栏下方看到完整地图区域且无横向滚动。
- **SC-002**: 用户从下拉框选择任一预设公园后，95% 的操作在 1 秒内完成可见定位与标记更新。
- **SC-003**: 在四个预设公园切换验证中，100% 的最终地图状态与当前下拉选择一致。
- **SC-004**: 在连续快速切换（3 次及以上）场景下，100% 的验证样本最终展示结果与最后一次选择一致。
- **SC-005**: 地图不可用场景下，100% 的验证样本仍可见公园下拉控件与清晰提示信息。

## Assumptions

- 分析页面已存在并继续作为该地图功能的承载入口页面。
- 预设公园定位信息由产品侧确认并可作为本期固定配置使用。
- 本期范围仅支持四个预设公园，不包含用户自定义地点或搜索输入。
- 本期不要求跨会话记忆用户上次选择的公园。
- 地图底图服务可能受网络影响，系统仅需提供可理解提示与可恢复操作路径。
