# Feature Specification: 记录页记事本

**Feature Branch**: `007-records-notebook`  
**Created**: 2026-04-01  
**Status**: Draft  
**Input**: User description: "在“记录”页面新增一个与登录账号绑定的记事本功能，支持查看统计、创建、编辑、删除、定位填充、地图选点、未登录拦截与登录提醒。"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 管理个人观测记录 (Priority: P1)

作为已经登录的观鸟用户，我希望在“记录”页面直接看到自己的记事本列表、记录总数和鸟种总数，并且能新增或删除记录，这样我可以持续整理自己的个人观测内容。

**Why this priority**: 记录的查看、创建和删除是记事本功能的最核心价值；如果用户无法先完成这些基本操作，后续的编辑细节和登录拦截都没有实际意义。

**Independent Test**: 使用已登录账号进入“记录”页面，确认原有标题和注释仍保留；点击“新增记录”创建一条有效记录后，确认列表和统计同步更新；再删除该记录并确认统计回退。

**Acceptance Scenarios**:

1. **Given** 用户已登录且进入“记录”页面，**When** 页面完成渲染，**Then** 页面保留“个人空间”“个人观测记录”标题及其注释，并在其下方显示记事本区域。
2. **Given** 用户已登录且已有若干记录，**When** 页面显示记事本区域，**Then** 顶部左侧显示“X 条记录 / Y 个鸟种”，下方有分割线，再下方是记录条目列表。
3. **Given** 用户已登录且点击页面底部居中的“新增记录”按钮，**When** 用户填写有效信息并点击“添加”，**Then** 系统保存新条目、关闭编辑区、刷新列表，并更新记录总数与鸟种总数。
4. **Given** 用户已登录且列表中存在一条记录，**When** 用户点击该条目右上角垃圾桶图标，**Then** 系统弹出“确认删除此记录？”确认框；若用户确认则删除该条目，若取消则保持原状并关闭确认框。
5. **Given** 用户已登录但暂无任何记录，**When** 页面显示记事本区域，**Then** 统计区显示 0 条记录 / 0 个鸟种，记录列表保持空状态且仍可通过“新增记录”开始创建。

---

### User Story 2 - 编辑完整观测条目 (Priority: P2)

作为已经登录的观鸟用户，我希望点击已有条目或新增条目时，能够在顶层编辑区里补充或修改时间、鸟名、鸟点和备注，并使用定位或地图辅助填写位置，这样我可以高效整理更完整的观测信息。

**Why this priority**: 仅有简单的增删还不足以形成可用的记事本；完整的时间、地点和备注编辑能力决定了记录是否真正具有长期参考价值。

**Independent Test**: 使用已登录账号打开一个已有条目或新建条目，确认编辑区在顶层弹出；分别测试日期选择、时间选择、鸟名填写、鸟点手动填写、定位填充、地图选点、备注字数限制、取消放弃编辑提示，并确认保存后条目内容正确更新。

**Acceptance Scenarios**:

1. **Given** 用户点击已有条目，**When** 编辑区在页面顶层弹出，**Then** 编辑区预填该条目的现有内容；若用户点击“新增记录”，则日期和时间默认填充为当前北京时间。
2. **Given** 编辑区已打开，**When** 用户打开日期选择器或时间选择器并选定值，再点击其他区域，**Then** 选择器关闭并将所选日期或时间填充到表单中。
3. **Given** 编辑区已打开，**When** 用户编辑鸟名、鸟点和备注，**Then** 系统允许用户直接输入文本，并将备注长度限制在 100 字以内。
4. **Given** 编辑区已打开，**When** 用户点击定位图标且设备位置可用，**Then** 系统自动获取当前位置并填充到鸟点字段；若设备位置不可用、被拒绝或超时，则系统保留当前草稿并显示明确提示。
5. **Given** 编辑区已打开，**When** 用户点击“地图”并完成地图选点，**Then** 系统将选定位置填充到鸟点字段；若用户取消选点，则保留原有字段内容不变。
6. **Given** 编辑区已打开且用户已修改内容，**When** 用户点击“取消”，**Then** 系统先弹出“是否放弃编辑？”确认提示；若用户确认则丢弃修改并关闭编辑区，若选择否则返回编辑区继续编辑。

### User Story 3 - 未登录时引导进入个人空间 (Priority: P3)

作为尚未登录的用户，我希望在尝试使用记事本功能时得到原地登录与提醒，而不是被强制跳转离开当前页面，这样我可以明确知道需要先完成登录或注册。

**Why this priority**: 记事本与个人账号绑定，未登录拦截是数据隔离与用户预期管理的重要部分；但它建立在已有记录交互能力之上，因此优先级低于核心记录流。

**Independent Test**: 在未登录状态进入“记录”页面并尝试点击“新增记录”或现有记事本交互入口，确认会原地弹出登录弹窗；输入不存在的助手账号时看到“请先注册助手账号！”，关闭或拒绝登录后继续看到“请登录个人空间！”的提醒弹窗，且不会跳转离开当前页面。

**Acceptance Scenarios**:

1. **Given** 用户未登录，**When** 用户尝试点击“新增记录”或其他记事本交互入口，**Then** 系统在当前页面原地弹出登录弹窗，而不是跳转到其他页面。
2. **Given** 用户通过记事本触发的登录弹窗输入一个不存在的助手账号，**When** 用户提交登录，**Then** 系统提示“请先注册助手账号！”并保持用户留在当前页面上下文中。
3. **Given** 用户未登录且关闭登录弹窗或未完成登录，**When** 记事本仍需要身份才能继续操作，**Then** 系统继续弹出“请登录个人空间！”提醒，并提供关闭弹窗选项而不是跳转到登录页面。
4. **Given** 用户已登录并保存过个人记录，**When** 用户刷新页面或稍后再次进入“记录”页面，**Then** 系统仍显示该账号自己的记事本内容，而不会显示其他账号的记录。

### Edge Cases

- 用户删除最后一条记录后，统计区如何立即更新为 0 条记录 / 0 个鸟种。
- 用户输入重复鸟名但不同时间或不同地点时，鸟种总数如何保持为去重后的数量。
- 用户在编辑区已填写部分内容时点击遮罩、返回或取消，系统如何保证先经过“是否放弃编辑？”确认而不是直接丢失草稿。
- 用户拒绝设备定位权限、设备无定位能力或定位超时时，系统如何保留手动填写能力并给出明确提示。
- 用户打开地图选点但未完成选点便关闭，系统如何避免覆盖现有鸟点文本。
- 用户在未登录状态下重复关闭登录提醒，系统如何继续保持记事本功能受限但不发生页面跳转。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统 MUST 在“记录”页面保留现有“个人空间”“个人观测记录”标题及其注释。
- **FR-002**: 系统 MUST 在标题下方显示记事本区域，并在区域顶部左侧显示“X 条记录 / Y 个鸟种”统计信息。
- **FR-003**: 系统 MUST 在统计信息下方显示分割线，并在分割线下方显示记录条目区。
- **FR-004**: 系统 MUST 让每条记录以北京时间展示时间、鸟点和鸟名，其中时间与鸟点位于条目左上区域，鸟名显示在下一行。
- **FR-005**: 系统 MUST 在每条记录右上角提供删除入口，并在删除前显示“确认删除此记录？”确认提示。
- **FR-006**: 系统 MUST 在用户确认删除后移除对应条目，并立即更新记录总数与鸟种总数；用户取消时不得修改任何记录数据。
- **FR-007**: 系统 MUST 在“记录”页面底部居中提供“新增记录”入口，并允许用户由此进入新条目编辑区。
- **FR-008**: 系统 MUST 在用户点击已有条目或“新增记录”时，于页面顶层弹出编辑区域而不是离开当前页面。
- **FR-009**: 系统 MUST 在新条目编辑区默认填充当前北京时间；已有条目编辑时 MUST 预填其现有内容。
- **FR-010**: 系统 MUST 提供分离的日期选择器和时间选择器，并在用户选定后于点击其他区域时关闭选择器并填充值。
- **FR-011**: 系统 MUST 提供可编辑的鸟名文本框和鸟点文本框，允许用户自行输入内容。
- **FR-012**: 系统 MUST 在鸟点字段附近提供设备定位入口；当当前位置获取成功时自动填充鸟点，失败时显示简体中文提示且保留当前草稿。
- **FR-013**: 系统 MUST 提供地图选点入口；用户完成选点后将选定位置填充到鸟点字段，取消选点时不得覆盖现有字段内容。
- **FR-014**: 系统 MUST 提供备注文本框，并将备注长度限制为最多 100 字。
- **FR-015**: 系统 MUST 在编辑区域底部提供“添加”和“取消”操作；点击“添加”时保存条目，点击“取消”时先弹出“是否放弃编辑？”确认提示。
- **FR-016**: 系统 MUST 阻止缺少必要信息的条目被保存，并向用户显示明确的简体中文提示。
- **FR-017**: 系统 MUST 将记事本内容与当前登录的助手账号绑定，并为每个账号维护独立的个人记事本数据。
- **FR-018**: 系统 MUST 将个人记事本数据保存在服务端的账号专属记录仓库中，并在页面刷新后继续为同一账号提供已保存的记录。
- **FR-019**: 系统 MUST 让记录总数反映当前账号的全部记录数量，让鸟种总数反映当前账号中去重后的鸟名数量。
- **FR-020**: 系统 MUST 在未登录用户尝试使用记事本功能时，在当前页面原地弹出登录弹窗，而不是跳转到其他页面。
- **FR-021**: 系统 MUST 在记事本触发的登录流程中，对不存在的助手账号显示“请先注册助手账号！”。
- **FR-022**: 系统 MUST 在用户关闭登录弹窗或未完成登录时，继续以当前页面弹窗方式提示“请登录个人空间！”，并提供关闭选项而不是跳转到登录页面。
- **FR-023**: 系统 MUST 保持登录用户的记事本仅对当前账号可见，不得在不同账号之间混用或泄露记录内容。
- **FR-024**: 记录页记事本相关的所有标题、标签、按钮、提示、确认文案、空状态、错误状态和占位文本 MUST 使用简体中文。

### API, UI, and Visualization Contract *(mandatory for web features)*

- **API-001**: 系统需要提供 `GET /api/records/notebook` 以供“记录”页面在进入、刷新和成功变更后获取当前登录账号的记事本摘要与条目列表；成功响应 MUST 返回统计信息、按显示顺序排列的记录条目，以及空状态所需提示；失败响应 MUST 至少区分未登录、数据不可用和账号上下文无效。
- **API-002**: 系统需要提供 `POST /api/records/notebook` 以供“新增记录”编辑区创建新条目；请求 MUST 包含时间、鸟名、鸟点和备注字段；成功响应 MUST 返回新条目与更新后的统计信息；失败响应 MUST 至少区分未登录、校验失败与保存失败。
- **API-003**: 系统需要提供 `PATCH /api/records/notebook/[recordId]` 以供已有条目的编辑区保存修改；成功响应 MUST 返回更新后的条目与统计信息；失败响应 MUST 至少区分未登录、记录不存在、校验失败与保存失败。
- **API-004**: 系统需要提供 `DELETE /api/records/notebook/[recordId]` 以供删除确认框删除单条记录；成功响应 MUST 返回删除结果与更新后的统计信息；失败响应 MUST 至少区分未登录、记录不存在与删除失败。
- **API-005**: 当前已有的登录能力在被记事本功能调用时 MUST 能区分“账号不存在”和“密码错误”两类失败结果，以支持“请先注册助手账号！”的原地提示。
- **UI-001**: “记录”页面需要新增一个记事本主区域，沿用现有标题与注释，并使用 Tailwind CSS + shadcn/ui 风格组件组合实现统计区、分割线、记录列表、空状态和底部“新增记录”入口。
- **UI-002**: 记录条目需要提供列表态、删除确认弹窗、空状态、加载状态和错误状态；编辑区域需要提供顶层弹窗态、日期选择器、时间选择器、输入框、备注区、定位入口、地图选点入口以及“添加 / 取消”操作。
- **UI-003**: 未登录用户触发记事本操作时，需要在当前页面原地显示登录弹窗与后续提醒弹窗，且关闭弹窗后仍保留当前页面上下文，不发生页面跳转。
- **UI-004**: 记录页相关可见文案 MUST 包含“X 条记录 / Y 个鸟种”“新增记录”“确认删除此记录？”“是否放弃编辑？”“请先注册助手账号！”“请登录个人空间！”以及定位、地图、备注、添加、取消等简体中文文案。
- **CHART-001**: `N/A`。本功能不包含图表或数据可视化需求。

### Key Entities *(include if feature involves data)*

- **Notebook Summary**: 当前登录账号的记事本统计信息，包含记录总数和去重后的鸟种总数。
- **Observation Record**: 单条观测记录，包含记录标识、所属助手账号、北京时间、鸟点、鸟名、备注以及显示排序所需信息。
- **Record Draft**: 用户在编辑区中尚未提交的记录草稿，包含日期、时间、鸟名、鸟点、备注以及定位或地图选点产生的待保存内容。
- **Notebook Access Prompt**: 未登录用户尝试使用记事本时触发的原地身份提醒状态，包含登录提示、账号不存在提示和继续登录提醒。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 95% 的已登录用户能够在进入“记录”页面后 2 秒内看到自己的记事本统计信息、记录条目区和“新增记录”入口。
- **SC-002**: 95% 的有效新增、编辑和删除操作能够在 2 秒内反映到当前页面的条目列表与统计信息上，且无需离开“记录”页面。
- **SC-003**: 100% 的已保存记录在页面刷新后仍能被同一助手账号重新看到，且不会出现在其他账号的个人记事本中。
- **SC-004**: 100% 的未登录记事本操作都会在当前页面上下文中触发登录或提醒弹窗，不发生强制跳转到其他页面。
- **SC-005**: 95% 的用户能够在 90 秒内完成一条包含时间、鸟名、鸟点和备注的观测记录创建或编辑。

## Assumptions

- 现有“记录”页面的标题与注释文案保持不变，仅在其下方扩展记事本能力。
- 记录列表默认按北京时间从新到旧排序，便于用户先看到最近一次观测。
- 鸟名和鸟点属于保存记录所必需的信息；备注为可选信息，但若填写则最多 100 字。
- 新建记录时默认使用当前北京时间；编辑已有记录时优先展示该记录当前已保存的时间内容。
- 地图选点在当前移动网页内完成，并以单个可展示的鸟点文本结果回填到表单中。
- 账号专属记事本数据保存在服务端可持续访问的记录仓库中，因此同一账号后续再次进入页面时仍可访问既有内容。
- 记事本触发的身份校验将复用现有登录体系，但需要补充“账号不存在”的可识别反馈，以支撑“请先注册助手账号！”提示。
