# Feature Specification: 分析页记事本导出

**Feature Branch**: `010-download-bird-notes`  
**Created**: 2026-04-04  
**Status**: Draft  
**Input**: User description: "我要在“分析”页面新增一个能下载当前登录的账号记事本的下载功能。具体要求如下：1.入口为“保存记录到本地”，文本形式，点击可将记事本中的所有条目下载到默认下载途径。2.入口位置在“新增记录”按钮下方。3.字号大小与“记录自己的观测发现，并回看最近外出时留下的笔记。”相同。4.当不处于账号登录状态或处于账号登录状态但记事本没有条目时，该入口隐藏；处于账号登录状态且记事本有条目时，入口显示。5.下载的文件类型为.txt，文件命名为“xx的观鸟记录”（xx为记事本对应的账号），txt文件的内容如下：“序号 日期：YYYY-MM-DD（相应条目的日期）；时间： HH-MM（相应条目的时间）；鸟点：xxx（相应条目的鸟点）；鸟名：xxx（相应条目的鸟名）.”一行写一条记录，两条记录之间需要换行。"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 导出当前账号记录 (Priority: P1)

作为已登录且已有记事本记录的用户，我希望在“分析”页面直接点击“保存记录到本地”，把当前账号全部记录下载为一个 TXT 文件，便于离线保存与回看。

**Why this priority**: 下载导出是本需求的核心价值；若不能下载，功能即不可用。

**Independent Test**: 使用一个已登录且有记录的账号进入“分析”页面，点击“保存记录到本地”，应在默认下载目录生成一个 `.txt` 文件，并包含全部记录。

**Acceptance Scenarios**:

1. **Given** 用户已登录且记事本有至少 1 条记录，**When** 用户在“保存记录到本地”入口点击一次，**Then** 系统下载一个 `.txt` 文件到默认下载路径。
2. **Given** 用户已登录且记事本有多条记录，**When** 用户下载成功后打开文件，**Then** 文件内容按“一行一条记录”展示，条目之间以换行分隔。

---

### User Story 2 - 控制入口显示条件 (Priority: P2)

作为用户，我希望“保存记录到本地”只在有意义时出现，避免在未登录或无记录时看到无效入口。

**Why this priority**: 入口可见性直接影响体验一致性和误操作率，但不高于核心下载能力。

**Independent Test**: 分别在未登录、已登录无记录、已登录有记录三种状态下进入“分析”页面，验证入口显隐是否符合要求。

**Acceptance Scenarios**:

1. **Given** 用户未登录，**When** 页面渲染完成，**Then** 不显示“保存记录到本地”入口。
2. **Given** 用户已登录但记事本记录为空，**When** 页面渲染完成，**Then** 不显示“保存记录到本地”入口。
3. **Given** 用户已登录且记事本记录非空，**When** 页面渲染完成，**Then** 在“新增记录”按钮下方显示“保存记录到本地”文本入口。

---

### User Story 3 - 保持入口样式与文本规范 (Priority: P3)

作为用户，我希望新增入口在视觉上与当前页面文案风格一致，下载文件命名和内容格式固定，便于识别和归档。

**Why this priority**: 样式与格式规范提升一致性和可读性，属于功能完善项。

**Independent Test**: 在已登录且有记录状态下，验证入口字号是否与“记录自己的观测发现，并回看最近外出时留下的笔记。”一致；验证下载文件名和每行文本格式。

**Acceptance Scenarios**:

1. **Given** 入口可见，**When** 用户查看入口样式，**Then** 入口字号与说明文案“记录自己的观测发现，并回看最近外出时留下的笔记。”一致。
2. **Given** 下载已完成，**When** 用户查看文件名，**Then** 文件名格式为“xx的观鸟记录.txt”，其中 `xx` 为当前账号名。
3. **Given** 下载已完成，**When** 用户查看任意一条导出行，**Then** 内容格式为“序号 日期：YYYY-MM-DD；时间：HH-MM；鸟点：xxx；鸟名：xxx。”。

---

### Edge Cases

- 当前账号名称包含文件名非法字符时，导出文件名如何保证可保存且可识别。
- 记录数较多时（例如 500 条以上）点击下载，是否仍能一次性导出全部条目且不丢行。
- 记录字段含前后空格时，导出文本是否应按页面显示值输出，避免出现异常空白。
- 同一用户短时间连续点击入口时，是否每次都能生成完整文件且不混入其他账号数据。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统 MUST 在“分析”页面提供“保存记录到本地”文本入口，用于导出当前登录账号的记事本记录。
- **FR-002**: 系统 MUST 将“保存记录到本地”入口放置在“新增记录”按钮下方。
- **FR-003**: 系统 MUST 使“保存记录到本地”入口字号与“记录自己的观测发现，并回看最近外出时留下的笔记。”文案字号一致。
- **FR-004**: 系统 MUST 在“未登录”状态隐藏“保存记录到本地”入口。
- **FR-005**: 系统 MUST 在“已登录但记事本无条目”状态隐藏“保存记录到本地”入口。
- **FR-006**: 系统 MUST 在“已登录且记事本有条目”状态显示“保存记录到本地”入口。
- **FR-007**: 系统 MUST 在用户点击入口后，将当前账号记事本的全部条目下载为 `.txt` 文件到默认下载路径。
- **FR-008**: 系统 MUST 使用“xx的观鸟记录.txt”作为下载文件名，其中 `xx` 为记事本所属账号名称。
- **FR-009**: 系统 MUST 使导出文件按“一行一条记录”输出，且记录之间以换行分隔。
- **FR-010**: 系统 MUST 使每条导出记录遵循以下字段顺序与文案结构：`序号 日期：YYYY-MM-DD；时间：HH-MM；鸟点：xxx；鸟名：xxx。`
- **FR-011**: 系统 MUST 保证导出的记录仅来自当前登录账号，不得包含其他账号数据。

### API, UI, and Visualization Contract *(mandatory for web features)*

- **API-001**: 本功能不新增 Route Handler；继续使用现有记事本读取能力返回当前登录账号的记录集合，并由页面发起导出。
- **UI-001**: 在“分析”页面对应的记事本区域新增“保存记录到本地”文本入口，入口位于“新增记录”按钮下方，保持现有页面结构不变。
- **UI-002**: 入口显示状态必须覆盖三种情况：未登录隐藏、已登录且无条目隐藏、已登录且有条目显示；下载行为需保持当前页面停留不变。
- **UI-003**: 页面新增和调整的用户可见文案使用简体中文，包括入口文本“保存记录到本地”及导出文本中的中文标签（日期、时间、鸟点、鸟名）。
- **CHART-001**: `N/A`。本功能不涉及图表或可视化。

### Key Entities *(include if feature involves data)*

- **导出文件（Bird Notes Export）**: 当前登录账号对应的一次性文本导出结果，包含文件名、文件类型和逐行记录文本。
- **导出记录行（Export Line Item）**: 从单条记事本记录映射出的文本行，包含序号、日期、时间、鸟点、鸟名。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 在“已登录且有条目”条件下，100% 的点击操作都能触发 `.txt` 文件下载。
- **SC-002**: 在未登录或无条目条件下，100% 的页面渲染都不显示“保存记录到本地”入口。
- **SC-003**: 100% 的导出文件名符合“xx的观鸟记录.txt”格式，且 `xx` 与当前账号一致。
- **SC-004**: 抽样导出的文件中，100% 的记录行都符合“序号/日期/时间/鸟点/鸟名”的固定顺序与文案格式。

## Assumptions

- 记事本条目以页面当前账号上下文为准，不跨账号合并导出。
- 记录中的日期来源于 `YYYY-MM-DD`，时间来源于 `HH:mm`，导出时按需求展示为 `HH-MM`。
- 导出内容以当前页面已加载的记事本数据为准，点击时无需用户二次确认。
- 浏览器环境支持标准文件下载行为，并使用系统默认下载路径保存文件。
