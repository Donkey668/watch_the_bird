# Feature Specification: 栖息地鸟种参考

**Feature Branch**: `005-park-species-reference`  
**Created**: 2026-04-01  
**Status**: Draft  
**Input**: User description: "我要在“分析”页面新增一个“栖息地鸟种参考”组件。位置：现有的已开发组件下方。功能：1.读取预设的相应公园的Excel列表，列表内容是所选公园的近年鸟类观测记录；2.展示方式：纵向滚动卡片列表，每个条目展示excel文件中对应的序号、鸟种名称、居留类型、保护级别。交互：每张卡片支持点击，点击后弹出位于图层顶部的模态框，展示该条目对应的“生态特征”和“观测难度”（二者间换行）；3.若列表条数大于10，最多读取并展示前10个条目，然后在组件底部生成一个“点击查看全部信息”按钮，点击后完整加载excel表格；4.列表滚动添加简单的惯性滚动/吸附动画，提升操作流畅度。"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 查看当前公园的鸟种参考预览 (Priority: P1)

作为分析页用户，我希望在现有分析组件下方直接看到当前公园近年鸟类观测记录的鸟种参考预览，这样我无需离开分析页就能快速了解该公园常见的鸟种线索。

**Why this priority**: 这是该需求的核心价值。若用户不能直接看到与当前公园对应的鸟种参考列表，这个组件就失去了存在意义。

**Independent Test**: 打开分析页并保持某个预设公园选中，确认页面在现有组件下方出现“栖息地鸟种参考”区域，并默认展示该公园记录中的前 10 条或全部可用条目。

**Acceptance Scenarios**:

1. **Given** 用户进入分析页且当前已有选中的预设公园，**When** 页面完成首轮数据加载，**Then** 现有已开发组件下方显示“栖息地鸟种参考”组件。
2. **Given** 当前公园存在近年鸟类观测记录，**When** 鸟种参考列表渲染完成，**Then** 每条卡片都展示序号、鸟种名称、居留类型、保护级别四项信息。
3. **Given** 当前公园对应的记录总数大于 10 条，**When** 组件首次加载成功，**Then** 页面默认仅展示前 10 条记录，并在组件底部显示“点击查看全部信息”按钮。

---

### User Story 2 - 查看单条鸟种的详细说明 (Priority: P2)

作为分析页用户，我希望点击任意鸟种卡片后能立即看到该条记录的生态特征和观测难度，这样我能快速判断该鸟种的观察价值与识别难度。

**Why this priority**: 列表预览只能提供快速浏览价值，真正支持用户判断和学习的是每条记录的补充说明信息。

**Independent Test**: 在鸟种参考列表中点击任意一张卡片，确认页面会弹出位于最上层的模态框，并按换行分开展示该条目对应的生态特征和观测难度。

**Acceptance Scenarios**:

1. **Given** 鸟种参考列表中至少存在一条记录，**When** 用户点击某条卡片，**Then** 页面弹出位于图层顶部的详情模态框。
2. **Given** 详情模态框已打开，**When** 用户查看内容，**Then** 模态框展示该条目的“生态特征”和“观测难度”，且二者在视觉上分行显示。
3. **Given** 详情模态框已打开，**When** 用户点击关闭按钮、遮罩层或执行等效关闭操作，**Then** 模态框关闭且列表原有滚动位置保留。

---

### User Story 3 - 按需完整查看鸟种记录并保持可读体验 (Priority: P3)

作为正在深入了解某个公园鸟况的用户，我希望在需要时再完整展开全部鸟种记录，同时在滚动浏览时仍保持顺滑、清晰的阅读体验，这样我可以兼顾首屏效率与深度查阅。

**Why this priority**: 预览和完整数据需要平衡加载效率与信息完整性；如果没有按需展开和顺滑滚动，用户要么首屏信息过载，要么难以深入查看全部记录。

**Independent Test**: 对一个记录数超过 10 条的公园，先确认组件只展示前 10 条并出现“点击查看全部信息”按钮；点击按钮后确认全部记录被加载并显示，且列表保持纵向滚动与顺滑吸附体验。

**Acceptance Scenarios**:

1. **Given** 当前公园的记录总数超过 10 条，**When** 用户点击“点击查看全部信息”按钮，**Then** 组件完整加载并展示该 JSON 数据源中的全部可用条目。
2. **Given** 当前公园的记录总数不超过 10 条，**When** 组件完成加载，**Then** 页面不显示“点击查看全部信息”按钮。
3. **Given** 用户在纵向滚动鸟种卡片列表，**When** 滚动停止，**Then** 列表保持轻量的惯性滚动与吸附式停靠效果，帮助用户更稳定地浏览卡片内容。

### Edge Cases

- 当前选中的预设公园没有对应的 JSON 数据源时，组件如何给出明确的空状态或不可用提示？
- 当前公园的 JSON 文件存在但没有任何有效记录时，组件如何避免显示空白列表？
- 某条记录缺少“生态特征”或“观测难度”字段时，详情模态框如何明确说明缺失内容而不是渲染空白？
- 用户在“点击查看全部信息”后立即切换到另一个公园时，系统如何避免把上一个公园的完整列表误显示到新公园下？
- 记录总数恰好为 10 条时，组件如何保证不额外显示“点击查看全部信息”按钮？

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统 MUST 在分析页现有已开发组件下方新增一个名为“栖息地鸟种参考”的内容区域。
- **FR-002**: 系统 MUST 将当前分析页已选中的预设公园作为唯一上下文，并据此读取对应公园的近年鸟类观测记录数据源。
- **FR-003**: 系统 MUST 支持从每个预设公园预设绑定的 JSON 列表中读取鸟类观测记录。
- **FR-004**: 系统 MUST 在组件首次加载时最多展示前 10 条有效记录；若总记录数不足 10 条，则展示全部可用记录。
- **FR-005**: 系统 MUST 为每条列表卡片展示序号、鸟种名称、居留类型、保护级别四项信息。
- **FR-006**: 用户 MUST 能够点击任意鸟种卡片以查看该条记录的详细信息。
- **FR-007**: 系统 MUST 在点击卡片后展示位于页面最上层的模态框，并在模态框中分行展示“生态特征”和“观测难度”。
- **FR-008**: 当当前公园记录总数大于 10 条时，系统 MUST 在组件底部显示“点击查看全部信息”按钮。
- **FR-009**: 当用户点击“点击查看全部信息”按钮后，系统 MUST 完整加载并展示该公园 JSON 列表中的全部可用记录。
- **FR-010**: 当当前公园记录总数小于或等于 10 条时，系统 MUST NOT 显示“点击查看全部信息”按钮。
- **FR-011**: 当用户切换预设公园时，系统 MUST 自动刷新鸟种参考列表，并确保最终显示结果只属于最后一次选中的公园。
- **FR-012**: 鸟种参考列表 MUST 以纵向滚动卡片列表形式展示，并提供轻量的惯性滚动与吸附式停靠体验。
- **FR-013**: 系统 MUST 为组件提供明确的加载中、空数据、数据源不可用和完整加载中状态，不得以空白区域代替反馈。
- **FR-014**: 当记录字段缺失时，系统 MUST 继续展示其余可用字段，并以明确文案说明缺失的详情信息。
- **FR-015**: 组件 MUST 适配目标手机竖屏宽度，并在预览、完整加载、空状态和错误状态下均不引入水平滚动。
- **FR-016**: 该组件的所有用户可见文案 MUST 使用简体中文。
- **FR-017**: 该功能 MUST 保持在分析页内部完成，不新增新的顶层导航页面或脱离当前公园上下文的独立流程。

### API, UI, and Visualization Contract *(mandatory for web features)*

- **API-001**: 需要一个分析页专用 Route Handler：`GET /api/analysis/habitat-species-reference?parkId={parkId}&view={preview|full}`。调用方为分析页中的“栖息地鸟种参考”组件。成功响应 MUST 至少包含当前公园标识、当前公园名称、总记录数、本次返回条数、是否已完整加载、记录列表，以及数据源可用状态；失败响应 MUST 至少区分无效公园、数据源缺失或无法读取三类错误。
- **API-002**: 成功响应中的每条记录 MUST 至少覆盖序号、鸟种名称、居留类型、保护级别、生态特征、观测难度六项字段，并允许对缺失详情字段显式标记为不可用。
- **API-003**: 当 `view=preview` 时，接口 MUST 最多返回前 10 条有效记录并告知总记录数；当 `view=full` 时，接口 MUST 返回该公园全部可用记录。
- **UI-001**: 分析页需要在现有分析组件之后新增一个“栖息地鸟种参考”组件，该组件由 Tailwind CSS + shadcn/ui 组合实现，至少包含外层卡片容器、纵向滚动列表、条目卡片、详情模态框、底部按钮和必要的状态提示。
- **UI-002**: 成功状态下，组件 MUST 优先显示可纵向滚动的卡片列表；每条卡片的主要信息必须在手机竖屏宽度内可直接扫描，不依赖横向滚动。
- **UI-003**: 详情模态框 MUST 位于图层顶部，打开后不得被页面顶部栏或其他悬浮元素遮挡，并能清晰展示“生态特征”和“观测难度”两行内容。
- **UI-004**: 当组件处于预览态且记录数超过 10 条时，底部 MUST 显示“点击查看全部信息”按钮；完整加载完成后，该按钮 MUST 隐藏或转化为已完成状态，不得继续误导用户重复加载。
- **UI-005**: 组件 MUST 提供与手机触控滚动匹配的轻量惯性滚动和吸附体验，但不得阻止用户自然上下滑动整个页面。
- **UI-006**: 组件中的标题、按钮、空状态、错误提示、详情标签和占位文案 MUST 全部使用简体中文。
- **CHART-001**: `N/A`。本功能为记录列表和详情模态框展示，不包含图表可视化需求。

### Key Entities *(include if feature involves data)*

- **Park Species Source**: 与某个预设公园绑定的鸟类观测记录来源，代表该公园近年观测记录对应的数据清单。
- **Bird Species Record**: 鸟种参考列表中的单条记录，至少包含序号、鸟种名称、居留类型、保护级别、生态特征和观测难度。
- **Species Reference Preview**: 面向首屏展示的鸟种记录预览集合，包含当前公园前 10 条或全部可用记录、总记录数和是否可继续完整加载。
- **Species Reference Detail**: 用户点击某条卡片后查看的单条鸟种详情，聚焦生态特征和观测难度两项补充信息。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 95% 的正常访问中，用户在进入分析页后 3 秒内可以看到当前公园对应的“栖息地鸟种参考”预览列表。
- **SC-002**: 100% 的预览列表成功渲染场景中，每条卡片都展示序号、鸟种名称、居留类型、保护级别四项基础信息。
- **SC-003**: 对于记录数超过 10 条的公园，100% 的首次加载场景都只展示前 10 条记录，并出现“点击查看全部信息”按钮。
- **SC-004**: 95% 的详情查看操作中，用户点击卡片后 1 秒内即可看到位于最上层的详情模态框。
- **SC-005**: 在目标手机宽度范围内，组件在预览、完整加载、空状态、错误状态下都不需要水平滚动，且纵向卡片浏览保持可连续完成。

## Assumptions

- 每个预设公园在本期范围内都可以映射到一个预设 JSON 记录源，且记录字段结构保持一致。
- JSON 列表中的“近年鸟类观测记录”已经过基础整理，可直接用于前端展示，不需要本期新增人工编辑流程。
- 本期不包含鸟种搜索、筛选、排序、分页导航或多维统计，仅覆盖预览、完整加载和详情查看。
- “点击查看全部信息”默认在当前组件内完成完整列表展开，不跳转到新的页面或外部文档。
- 当某些详情字段缺失时，系统以明确的不可用文案代替空白值，避免用户误解为渲染失败。
- 所有面向前端的数据访问都通过 App Router Route Handlers 暴露，浏览器不会直接读取本地 JSON 文件。
