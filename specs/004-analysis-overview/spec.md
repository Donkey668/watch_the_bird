# Feature Specification: 分析总览核心信息

**Feature Branch**: `004-analysis-overview`  
**Created**: 2026-04-01  
**Status**: Draft  
**Input**: User description: "我要完善网页中的\"观鸟指数分析-今日概览\"占位符。区域布局在天气与观鸟指数组件下方。区域名称确定为“分析总览”，去除“观鸟指数分析”“今日概览”这两个占位标题以及无意义的占位摘要。获取北京时间和日期，生成核心信息。核心信息说明如下：一. 栖息地活跃度对应说明：观鸟指数对应之前根据天气获取的值。1. 观鸟指数为适宜：06:00–09:00、09:00–12:00、14:00–16:00、16:00–18:00：较高；12:00–14:00：中等；18:00–次日06:00：较低。2. 观鸟指数为较适宜：06:00–09:00、16:00–18:00：较高；09:00–12:00、12:00–14:00、14:00–16:00：中等；18:00–次日06:00：较低。3. 观鸟指数为不适宜：所有时间段的栖息地活跃度均为较低。二. 迁徙信号对应说明：11、12、1、2月为极高；3、9、10月为较高；4、8月为中等；5、6、7月为较低。三. 观测可信度固定为稳定。补充澄清：北京时间显示格式为“2026年4月1日 09:30”样式；分析总览中不显式展示“当前时段”标签；栖息地活跃度暂不可用时显示“暂不可用”。"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 查看有意义的分析总览 (Priority: P1)

作为分析页用户，我希望在天气与观鸟指数区域下方看到一个真正可用的“分析总览”，这样我能在同一屏内快速理解当天此刻的观鸟参考信息，而不是阅读占位标题和无意义摘要。

**Why this priority**: 这是替换当前占位内容的核心价值。如果该区域仍然只是说明性占位文案，用户无法从分析页得到可直接参考的结论。

**Independent Test**: 打开分析页，在天气与观鸟指数区域下方确认只出现一个名为“分析总览”的区域，且区域内容为按“2026年4月1日 09:30”样式展示的北京时间和核心信号，而不是“观鸟指数分析”“今日概览”及其占位摘要。

**Acceptance Scenarios**:

1. **Given** 用户进入分析页且天气与观鸟指数区域已完成加载，**When** 用户继续向下查看，**Then** 天气与观鸟指数区域下方显示名为“分析总览”的内容区。
2. **Given** 分析总览区域已渲染，**When** 用户查看该区域标题与导语，**Then** 页面不再显示“观鸟指数分析”“今日概览”这两个占位标题，也不再显示无意义的占位摘要。
3. **Given** 分析总览区域成功显示，**When** 用户浏览内容，**Then** 用户可以看到按“YYYY年M月D日 HH:mm”格式展示的北京时间，以及栖息地活跃度、迁徙信号、观测可信度三项核心信息。

---

### User Story 2 - 基于当前时段与季节获得即时判断 (Priority: P2)

作为准备出门观鸟的用户，我希望系统结合当前北京时间、当前月份以及已有的当日观鸟指数，自动给出当前时段的栖息地活跃度和当月迁徙信号，这样我能更快判断是否值得立刻前往观测。

**Why this priority**: 该故事把原本静态的概览区转化为可行动的信息摘要，是分析页对当天观鸟决策产生帮助的关键。

**Independent Test**: 使用可控的北京时间和已有观鸟指数结果，逐一验证不同时间段和月份组合，确认分析总览中的栖息地活跃度和迁徙信号与规则完全一致。

**Acceptance Scenarios**:

1. **Given** 当前北京时间处于某个已定义时段，且当日观鸟指数为“适宜”，**When** 分析总览生成结果，**Then** 栖息地活跃度按该时段对应规则显示“较高”“中等”或“较低”。
2. **Given** 当前北京时间月份属于 11、12、1、2 月之一，**When** 分析总览生成迁徙信号，**Then** 迁徙信号显示为“极高”。
3. **Given** 观鸟指数为“不适宜”，**When** 分析总览生成栖息地活跃度，**Then** 不论当前处于哪个时段，栖息地活跃度均显示为“较低”。

---

### User Story 3 - 在边界和部分不可用场景下保持清晰 (Priority: P3)

作为分析页用户，我希望在时间边界、跨日时段或观鸟指数暂不可用时，分析总览仍然给出明确、一致的结果或不可用说明，这样我不会把错误的时段判断当作真实建议。

**Why this priority**: 该区域依赖时间规则和既有观鸟指数结果，若没有清晰的边界定义和降级行为，用户很容易在凌晨、整点切换或部分失败时看到误导性结果。

**Independent Test**: 验证 06:00、09:00、12:00、14:00、16:00、18:00 等边界时间点，以及观鸟指数暂不可用场景，确认分析总览始终给出唯一且可解释的状态。

**Acceptance Scenarios**:

1. **Given** 当前北京时间恰好位于 06:00、09:00、12:00、14:00、16:00 或 18:00 任一边界点，**When** 分析总览计算当前时段，**Then** 系统按照边界起始时刻所属的新时段生成唯一结果。
2. **Given** 当前北京时间位于 18:00 至次日 06:00 之间，**When** 分析总览计算栖息地活跃度，**Then** 系统将其视为夜间跨日时段并按规则返回对应结果。
3. **Given** 天气信息可见但当日观鸟指数暂不可用，**When** 分析总览渲染，**Then** 页面保留北京时间、迁徙信号和观测可信度，并对栖息地活跃度给出明确的暂不可用说明，而不是伪造一个等级。

### Edge Cases

- 当当前时间恰好落在 06:00、09:00、12:00、14:00、16:00、18:00 的边界点时，系统如何保证只命中一个时段且结果不冲突？
- 当北京时间处于 00:00 至 05:59 时，系统如何将其归入“18:00–次日06:00”的跨日时段？
- 当天气查询成功但观鸟指数暂不可用时，系统如何避免错误推导栖息地活跃度？
- 当月份处于 8 月、9 月等过渡期时，系统如何保证迁徙信号仍按固定映射返回唯一等级？

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统 MUST 在天气与观鸟指数组件下方提供一个名为“分析总览”的内容区域。
- **FR-002**: 系统 MUST 移除当前概览区中的“观鸟指数分析”“今日概览”占位标题以及无意义的占位摘要，不得继续以说明性占位文案替代真实内容。
- **FR-003**: 系统 MUST 以北京时间作为分析总览的唯一时间基准，并按“YYYY年M月D日 HH:mm”格式生成当前日期与当前时间上下文。
- **FR-004**: 系统 MUST 在分析总览中展示至少以下核心信息：北京时间日期时间、栖息地活跃度、迁徙信号、观测可信度。
- **FR-005**: 系统 MUST 将当前分析上下文中的当日观鸟指数作为栖息地活跃度的唯一指数输入，不得使用与当前天气结果无关的其他等级来源。
- **FR-006**: 系统 MUST 按以下北京时间分段生成当前时段：06:00–09:00、09:00–12:00、12:00–14:00、14:00–16:00、16:00–18:00、18:00–次日06:00。
- **FR-007**: 当观鸟指数为“适宜”时，系统 MUST 将 06:00–09:00、09:00–12:00、14:00–16:00、16:00–18:00 的栖息地活跃度判定为“较高”，12:00–14:00 判定为“中等”，18:00–次日06:00 判定为“较低”。
- **FR-008**: 当观鸟指数为“较适宜”时，系统 MUST 将 06:00–09:00、16:00–18:00 的栖息地活跃度判定为“较高”，09:00–12:00、12:00–14:00、14:00–16:00 判定为“中等”，18:00–次日06:00 判定为“较低”。
- **FR-009**: 当观鸟指数为“不适宜”时，系统 MUST 将所有时段的栖息地活跃度统一判定为“较低”。
- **FR-010**: 系统 MUST 按月份固定映射迁徙信号：11、12、1、2 月为“极高”；3、9、10 月为“较高”；4、8 月为“中等”；5、6、7 月为“较低”。
- **FR-011**: 系统 MUST 将观测可信度固定显示为“稳定”，且不得因时间、月份或公园切换产生变化。
- **FR-012**: 当北京时间处于规则边界时，系统 MUST 将边界起始时刻视为新时段的开始，并只返回一个唯一的栖息地活跃度结果。
- **FR-013**: 当北京时间处于 18:00 至次日 06:00 的跨日区间内时，系统 MUST 按夜间时段处理栖息地活跃度，而不因日期跨天产生第二套规则。
- **FR-014**: 当当日观鸟指数暂不可用时，系统 MUST 将栖息地活跃度显示为“暂不可用”，且 MUST 保留北京时间、迁徙信号与观测可信度展示。
- **FR-015**: 当用户切换公园导致天气与观鸟指数结果刷新时，系统 MUST 保证分析总览与当前选中的公园及其最新观鸟指数结果保持同步，不得展示旧公园残留结论。
- **FR-016**: 分析总览区域 MUST 适配目标手机竖屏宽度，并在成功、部分不可用与刷新状态下都不引入水平滚动。
- **FR-017**: 分析总览区域的用户可见文案 MUST 全部使用简体中文。
- **FR-018**: 分析总览区域 MUST NOT 显式展示“当前时段”标签或单独的时段名称；时段规则仅用于内部计算栖息地活跃度。

### API, UI, and Visualization Contract *(mandatory for web features)*

- **API-001**: 需要复用并扩展现有聚合型 Route Handler：`GET /api/analysis/birding-outlook?parkId={parkId}`。调用方为分析页中天气与观鸟指数区域下方的“分析总览”模块。成功响应 MUST 能提供当前公园上下文、当日观鸟指数、北京时间上下文和分析总览结果；失败或部分成功响应 MUST 能表达分析总览的可用性。
- **API-002**: 该接口返回的分析总览数据 MUST 至少包含按“YYYY年M月D日 HH:mm”格式可展示的北京时间、栖息地活跃度值或不可用状态、迁徙信号值、观测可信度值。
- **API-003**: 当天气信息可用但观鸟指数暂不可用时，接口 SHOULD 返回部分成功结果，使前端仍可渲染北京时间、迁徙信号和观测可信度，同时将栖息地活跃度标记为不可用。
- **UI-001**: 分析页需要在现有天气与观鸟指数组件下方新增或改造一个“分析总览”区域，该区域由 Tailwind CSS + shadcn/ui 组合实现，至少包含标题、北京时间上下文、核心信息行或卡片、分隔结构，以及必要的状态提示。
- **UI-002**: 分析总览的标题层级 MUST 只保留“分析总览”这一有效区域名称，不再保留“观鸟指数分析”“今日概览”及其占位摘要。
- **UI-003**: 成功状态下，分析总览 MUST 以易扫读方式突出展示栖息地活跃度、迁徙信号和观测可信度，并同时显示按“YYYY年M月D日 HH:mm”格式展示的北京时间作为解释上下文。
- **UI-004**: 分析总览 MUST NOT 在界面中显式展示“当前时段”标签。
- **UI-005**: 当栖息地活跃度暂不可用时，页面 MUST 直接将该项显示为“暂不可用”，且不得隐藏仍可展示的迁徙信号与观测可信度。
- **CHART-001**: `N/A`。本功能只补全分析总览文案与规则化信息，不引入图表。

### Key Entities *(include if feature involves data)*

- **Beijing Time Context**: 分析总览使用的北京时间上下文，包含按“YYYY年M月D日 HH:mm”格式展示的当前日期时间，用于驱动规则并作为界面解释信息。
- **Habitat Activity Summary**: 基于当前北京时间时段和当日观鸟指数生成的栖息地活跃度结果，取值为“较高”“中等”“较低”或明确的暂不可用状态。
- **Migration Signal Summary**: 基于当前北京时间月份映射出的迁徙信号结果，取值为“极高”“较高”“中等”“较低”。
- **Analysis Overview Snapshot**: 面向分析页展示的总览快照，聚合北京时间、栖息地活跃度、迁徙信号和固定的观测可信度。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 100% 的分析总览成功渲染场景中，页面只显示“分析总览”作为该区域名称，不再显示“观鸟指数分析”“今日概览”及其占位摘要。
- **SC-002**: 在覆盖 6 个时段和 6 个边界时刻的验证场景中，栖息地活跃度与时段规则、观鸟指数规则保持 100% 一致。
- **SC-003**: 在 12 个月份映射验证中，迁徙信号等级与月份规则保持 100% 一致。
- **SC-004**: 在天气与观鸟指数数据已准备好的正常访问场景中，用户能在 3 秒内看到包含北京时间和三项核心信息的分析总览。
- **SC-005**: 在目标手机宽度范围内，分析总览在成功、部分不可用和刷新状态下均不需要水平滚动。

## Assumptions

- 本功能沿用分析页当前已有的天气与观鸟指数结果作为输入，不引入新的独立分析来源。
- “北京时间”统一指中国标准时间，用作分析总览的日期、时间和月份判断基准，而不是用户设备所在时区。
- 本期迁徙信号仅依据固定月份映射生成，不接入额外的迁徙监测数据源。
- 当观鸟指数不可用时，迁徙信号和观测可信度仍可独立展示，因此分析总览不需要整体隐藏。
- 本功能继续保留在分析页内部完成，不新增新的顶层导航页面，也不引入图表需求。
